# AGENTS.md

## Cursor Cloud specific instructions

This is a **Nuxt 3** static content portfolio/blog (`@nuxt/content`, `@nuxt/ui`, Tailwind). Package manager is **yarn** (`yarn.lock`); use `yarn`, not npm/pnpm.

- Start the dev server with `yarn dev` (serves on http://localhost:3000). SSR is enabled. `postinstall` runs `nuxt prepare` to generate `.nuxt` types.
- There is **no test script and no ESLint config**. The only "lint" available is Prettier: `npx prettier --check "**/*.{ts,vue,js}"`. Note that several existing files already report Prettier warnings on a clean checkout â€” do not treat those pre-existing warnings as regressions.
- `@nuxtjs/google-fonts` downloads the Inter font from Google Fonts during `nuxt prepare`/dev startup, so first install/build needs network access.
- Content pages are driven by files under `content/` (articles, projects, experiences, educations, uses, lab). Add/edit those Markdown/JSON files to change site content; `pages/` renders them.
- `nuxt.config.ts` sets `devtools.enabled: false`.

### Feature notes

- **Video Editing tab** (`pages/videos.vue` + `components/App/VideoPlayer.vue`): shows a YouTube playlist via the **IFrame Player API** (loaded lazily on first play). Config lives in `app.config.ts` under `video`: `playlistId` (full URL or bare `PL...` ID), optional `channelUrl` / `facebookUrl` buttons, and optional `videos` (array of YouTube URLs/11-char IDs) to curate the choices rail. The iframe uses YouTube's official `embed/videoseries` URL (optionally including the Shareâ†’Embed `?si=` token via `video.playlistSi`); the Player API is attached to that existing iframe (`enablejsapi=1`). When `videos` is empty the player reads the playlist live via `getPlaylist()` and builds the choices rail automatically (works for unlisted playlists, since unlisted still embeds â€” only fully private does not). Playback starts on a **random** video (`setShuffle` + `playVideoAt`); the choices thumbnails and "Shuffle" button re-select. Live playlist auto-population and playback require a real browser â€” see the bot-check note below.
- **Gotcha â€” Tailwind `aspect-video`:** under this `@nuxt/ui` Tailwind preset, the core `aspect-video` utility is not emitted into the served CSS (a container relying on it collapses to 0 height). Use an inline `style="aspect-ratio: 16 / 9"` instead (see `VideoPlayer.vue`).
- **Testing YouTube embeds in the cloud VM:** the datacenter IP triggers YouTube's "Sign in to confirm you're not a bot" screen inside embeds. This is an environment limitation, not an app bug â€” embeds render correctly on a normal user's browser.
- **Contact form** (`pages/contact.vue` + `server/api/contact.post.ts`): posts to the Nitro route `/api/contact`, which emails the message to `contactTo` via Gmail SMTP (nodemailer) when `NUXT_GMAIL_APP_PASSWORD` (and `NUXT_GMAIL_USER`/`gmailUser`, defaulted to `delf.boston@gmail.com`) are set â€” configured as `runtimeConfig` env vars locally and in Vercel. If Gmail creds are missing the route returns `503 email-not-configured` and the client falls back to the keyless FormSubmit relay (`app.config.ts` â†’ `contact.formEndpoint`), so the form still works without a backend. The server route requires a server/serverless deploy (e.g. Vercel `nuxt build`), not static `nuxt generate`. Note: adding a brand-new file under `server/` requires a dev-server restart to register the route.
- **Images / `sharp`:** `NuxtImg` (`@nuxt/image` â†’ IPX) optimizes images at request/build time via `sharp`. `sharp` is pinned to `0.33.5` via a `resolutions` entry in `package.json` because the transitive default (`0.32.6`) fails to self-register on Node 22 ("Module did not self-register"), which makes every optimized image 500 and render blank. If IPX image URLs (`/_ipx/...`) start returning 500 after a dependency change, check the installed `sharp` version first. Plain `<img>`/`UAvatar` thumbnails bypass IPX and are unaffected.
